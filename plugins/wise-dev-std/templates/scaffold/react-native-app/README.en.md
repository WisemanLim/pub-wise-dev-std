| Layer | Tool |
|-------|------|

## Prerequisites

- Node.js 22+
- pnpm 9+ (`npm i -g pnpm`)
- Expo CLI (`pnpm add -g expo-cli` or `npx expo`)
- EAS CLI (`pnpm add -g eas-cli`)
- Xcode 16+ _(macOS required for iOS simulator/device builds)_
- Android Studio _(for Android emulator)_

```bash
# Login to EAS
eas login
```

## Quick Start

```bash
# 1. Install dependencies
pnpm install

# 2. Start Expo dev client (Metro bundler)
make dev
# → Scan QR code with Expo Go app (iOS/Android)
```

## Make Targets

| Target | Description |
|--------|-------------|
| `make dev` | Start Expo dev server (`pnpm expo start`) |
| `make ios` | Run on iOS simulator (`pnpm expo run:ios`) |
| `make android` | Run on Android emulator (`pnpm expo run:android`) |
| `make test` | Jest tests + TypeScript check |
| `make build` | EAS all-platform build (`eas build -p all --profile production`) |
| `make deploy` | EAS all-platform submit (`eas submit -p all`) |

### Direct EAS Usage

```bash
# build by profile:
eas build -p ios --profile preview       # internal testing
eas build -p android --profile staging

# OTA update:
eas update --branch production --message "hotfix"

# store submit:
eas submit -p ios
eas submit -p android
```

## Environment Variables

| File | Purpose |
|------|---------|
| `.env.local` | Local development (loaded via Expo `extra`) |
| `.env.dev` | Dev server |
| `.env.staging` | Staging |
| `.env.prod` | Production |

Environment variables are loaded in `app.json` / `app.config.ts` via the `extra` field.

> **Warning**: Never commit API keys or secrets. Use EAS Secrets instead.

## Build & Deploy

### EAS Build

```bash
make build
# or per-platform:
eas build -p ios --profile production
eas build -p android --profile production
```

### Store Submit

```bash
make deploy
# or:
eas submit -p ios     # App Store Connect
eas submit -p android # Play Store
```

### OTA Update (JS bundle only)

```bash
eas update --branch production --message "bug fix"
```

> **Push secrets**: `eas secret:push --env-file .env.prod`
