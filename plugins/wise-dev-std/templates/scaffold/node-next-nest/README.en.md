| Container | Docker / Docker Compose v2 |
| Deploy | Helm 3 (Kubernetes) |
| CI/CD | GitHub Actions |

## Prerequisites

- Node.js 22+
- pnpm 9+ (`npm i -g pnpm`)
- Docker Desktop (or Docker Engine + Compose plugin)
- GNU Make

## Quick Start

```bash
# 1. Install dependencies
pnpm install

# 2. Copy environment file
cp .env.local .env

# 3. Start infrastructure (Postgres :5432 + Redis :6379)
make up

# 4. Run all workspace dev servers
make dev
# → Next.js: http://localhost:3000
# → NestJS:  http://localhost:4000
```

> **Note**: `docker-compose.yml` defines infra only (Postgres/Redis).  
> Apps (web/api) run on the host to share the pnpm workspace lockfile.

## Make Targets

```
make <target> [ENV=<env>]
```

| Target | Description |
|--------|-------------|
| `make up` | Start PostgreSQL + Redis containers |
| `make down` | Stop and remove all containers |
| `make dev` | Run all workspace dev servers, foreground (`pnpm -r dev`) |
| `make run` | Start web+api as **PM2** daemons (`pm2 start ecosystem.config.cjs`) |
| `make stop` | Stop & delete PM2 processes |
| `make restart` | Restart PM2 processes |
| `make logs` | Tail PM2 aggregated logs |
| `make ps` | PM2 process status (`pm2 ls`) |
| `make test` | Run all workspace tests (`pnpm -r test`) |
| `make build` | Build all workspace apps (`pnpm -r build`) |
| `make deploy` | Deploy to Kubernetes via Helm |

### Local multi-process (PM2)

For host runs, manage web (Next) + api (Nest) together. Apps are defined in `ecosystem.config.cjs`
(add workers as needed). PM2 ships as a `devDependency`, usable right after `pnpm install`, and the
same config can drive bare-metal prod.

```bash
make run      # start daemons (background)
make logs     # tail logs
make ps       # status
make stop     # stop
```

### Target a specific app

```bash
pnpm --filter web dev        # Next.js only
pnpm --filter api dev        # NestJS only
pnpm --filter api test:e2e   # e2e tests
```

## Environment Variables

| File | Purpose |
|------|---------|
| `.env.local` | Local development (exclude from Git) |
| `.env.dev` | Dev server |
| `.env.staging` | Staging |
| `.env.prod` | Production (manage secrets via CI/Vault) |

Key variables:

| Variable | Default | Description |
|----------|---------|-------------|
| `POSTGRES_USER` | `app` | DB user |
| `POSTGRES_PASSWORD` | `app` | DB password |
| `POSTGRES_DB` | `app` | DB name |
| `DATABASE_URL` | — | Full connection string |

> **Warning**: Never commit real secrets to `.env.prod`.

## Build & Deploy

### Local Build

```bash
make build
# individual apps:
pnpm --filter web build
pnpm --filter api build
```

### Docker Production Build

```bash
# multi-stage build from root context:
docker build -f apps/api/Dockerfile -t {{PROJECT_NAME}}-api .
docker build -f apps/web/Dockerfile -t {{PROJECT_NAME}}-web .
```

### Kubernetes (Helm)

```bash
make deploy
# or:
helm upgrade --install {{PROJECT_NAME}} ./deploy/helm \
  --set api.image.tag=$(git rev-parse --short HEAD) \
  --set web.image.tag=$(git rev-parse --short HEAD)
```
